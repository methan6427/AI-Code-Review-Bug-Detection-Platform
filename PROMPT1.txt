Implement a focused dashboard UX improvement pass for the AI Code Review Platform frontend.

Goal:
Make the dashboard more useful, interactive, and product-like without changing architecture or breaking routes.

Tasks:
1. Make dashboard metric cards clickable where appropriate:
   - Repositories -> /repositories
   - Recent scans -> /scans
   - Open issues -> relevant issue/scan view if available
   - Critical issues -> critical-severity filtered view if available
2. Add a compact Quick Actions section with actions like:
   - Run manual scan
   - Add repository
   - Install GitHub App
   - Import repositories
3. Add a workspace health summary block:
   - overall state label such as Stable / Needs attention
   - repositories monitored
   - last scan time
   - open issues count
4. Add a recent activity feed showing meaningful recent events if data is available, otherwise create a graceful placeholder state.
5. Add small trend or status indicators where helpful, but keep them simple and subtle.
6. Keep the current dark SaaS design language and make everything cohesive.

Constraints:
- Do not rewrite routes or backend behavior
- Do not add heavy new dependencies
- Prefer reusable components
- Keep interactions accessible and obvious

At the end:
1. summarize what was improved
2. list files changed
3. note any dashboard items that still need backend support
