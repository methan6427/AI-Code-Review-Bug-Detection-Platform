Implement an issue triage UX improvement pass for the AI Code Review Platform frontend.

Goal:
Make issue cards feel clearer, more actionable, and more professional.

Tasks:
1. Standardize the severity color system for:
   - critical
   - high
   - medium
   - low
   - info
2. Add category icons where appropriate using the current icon system.
3. Improve issue card hierarchy:
   - clearer title emphasis
   - better file/rule layout
   - clearer recommendation section
4. Improve issue action grouping:
   - Resolve
   - Ignore
   - Reopen
   - optionally False positive if already supported or safe to add visually only
5. If practical, add grouping or organization improvements by file or severity.
6. Add a compact code/location preview if data supports it.
7. Preserve current behavior while making the UI feel more polished and easier to scan.

Constraints:
- Do not change triage backend logic
- Keep actions obvious and accessible

At the end:
1. summarize issue UX changes
2. list files changed
3. mention any issue-related UX improvements left for future work
