Implement a scan monitoring UX improvement pass for the AI Code Review Platform frontend.

Goal:
Make scans easier to read and compare at a glance, especially across manual scans, GitHub push scans, and pull request scans.

Tasks:
1. Improve scan status presentation with stronger visual states for:
   - completed
   - running
   - failed
   - retrying
   - queued
2. Introduce a consistent scan source badge or icon system for:
   - Manual
   - GitHub Push
   - Pull Request
3. Improve scans list readability:
   - clearer hierarchy for title, source, branch, and commit data
   - better spacing and grouping of metadata
   - more obvious clickable rows or cards
4. Surface scan performance metrics where data exists:
   - duration
   - files analyzed
   - issues detected
5. Make changed-file and execution context easier to understand in both list and detail entry points.
6. Preserve the current dark SaaS product styling and keep the result restrained rather than flashy.

Constraints:
- Do not change the current scans API or routes
- Prefer clarity and reuse over adding decorative UI
- Do not add heavy new dependencies

At the end:
1. summarize the scan monitoring UX improvements
2. list files changed
3. note any scan fields or metrics that still need backend support
