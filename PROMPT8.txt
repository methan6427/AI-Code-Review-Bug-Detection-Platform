Implement a product polish pass for notifications and user feedback in the AI Code Review Platform frontend.

Goal:
Make user actions feel acknowledged and reduce ambiguity after important operations.

Tasks:
1. Audit existing feedback patterns for actions like:
   - repository added
   - repository import success/failure
   - scan queued
   - issue triaged
   - GitHub connected
   - auth errors
2. Add or improve toast/inline feedback where helpful.
3. Standardize success, warning, and error messaging style.
4. Improve placement and timing so feedback is noticeable without feeling noisy.
5. Keep the tone product-like, concise, and developer-oriented.
6. Reuse the existing notification system if one already exists.

Constraints:
- Do not add a large new notification dependency
- Keep it consistent with the design system
- Do not change backend behavior or route structure

At the end:
1. summarize feedback improvements
2. list files changed
3. note any actions that still lack good user feedback
