Implement a repository UX improvement pass for the AI Code Review Platform frontend.

Goal:
Make repositories feel like first-class product entities with better clarity, state communication, and actionability.

Tasks:
1. Improve repository cards with clearer status chips such as:
   - GitHub App connected
   - Manual
   - Ready to scan
   - Last synced/imported if supported
2. Add subtle repository icons where appropriate using the existing icon system.
3. Add a simple repository health/state indicator if data allows.
4. Build or improve a repository details hero section that clearly shows:
   - repo name
   - owner
   - branch
   - source type
   - GitHub/App connection status
   - last scan
   - primary actions
5. Ensure a clear manual scan trigger/action exists on repository details.
6. Improve repository card hover/click affordance and metadata hierarchy.
7. Preserve the existing dark theme and current design direction.

Constraints:
- Do not rewrite backend flows
- Do not break repository creation/import
- Keep styling maintainable and consistent

At the end:
1. summarize repository UX changes
2. list files changed
3. note any repository state fields that would benefit from future backend expansion
